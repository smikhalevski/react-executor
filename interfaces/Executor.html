Interface Executor<Value>

Manages the async task execution process and provides ways to access task execution results, abort or replace the task execution, and subscribe to its state changes.

interface Executor<Value> {
    isFulfilled: boolean;
    isInvalidated: boolean;
    isPending: boolean;
    isRejected: boolean;
    isSettled: boolean;
    key: string;
    reason: any;
    value: undefined | Value;
    abort(reason?): this;
    clear(): this;
    execute(task): AbortablePromise<Value>;
    getOrDefault(defaultValue): Value;
    getOrThrow(): Value;
    getOrWait(): AbortablePromise<Value>;
    invalidate(): this;
    reject(reason): this;
    resolve(value): this;
    retry(): AbortablePromise<Value>;
    subscribe(listener): (() => void);
}

Methods

abort

  • abort(reason?): this

  • Instantly aborts pending execution and preserves available results as is. Value (or error) returned from pending task callback is ignored. The signal passed to the executed task callback is aborted.

    Parameters

    • Optional reason: unknown

      The abort reason passed to the pending promise.

    Returns this

clear

  • clear(): this

  • Clears available results and doesn't affect the pending execution. Executor can still be retried after being cleared.

    Returns this

execute

  • execute(task): AbortablePromise<Value>

  • Executes a task and populates the executor with the returned result.

    Instantly aborts pending execution (if any), marks executor as pending and then invokes the task callback.

    If other execution was started before the promise returned by the task callback is fulfilled then the signal is aborted and the returned result is ignored.

    Parameters

    • task: Task<Value>

      The task callback that returns the new result for the executor to store.

    Returns AbortablePromise<Value>

    The promise that is resolved with the result of the task.
